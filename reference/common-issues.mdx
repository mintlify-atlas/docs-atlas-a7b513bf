This page collects common issues and their solutions. Before reporting a problem on GitHub, check whether it appears here and gather diagnostics.

## Collecting Diagnostics

<Note>
The plugin has built-in diagnostics collection under Plugin → Collect Diagnostics. Clicking the button copies a report to the clipboard that you can paste into a GitHub issue. Private information like usernames is anonymized.
</Note>

### Log Files

Log files are stored in the `.logs` subfolder of the plugin's user data directory. You can open this folder using the "View log files" link in the plugin's connection settings.

Typical paths:

| OS | Path |
|----|------|
| Windows | `C:\Users\<name>\AppData\Roaming\krita\ai_diffusion\` |
| Linux | `~/.local/share/krita/ai_diffusion/` |
| macOS | `~/Library/Application Support/krita/ai_diffusion/` |

### ComfyUI Workflow Export

You can export the ComfyUI workflow JSON generated by the plugin. Enable "Dump Workflow" in Interface settings. The exported `workflow.json` is saved in the same folder as the log files.

## Installation Issues

<AccordionGroup>
  <Accordion title="Plugin is grayed out in the Python Plugins Manager">
    The most common cause is installing from GitHub's "Download ZIP" button, which generates a source-only archive that is missing required files. Download the correct package from the [Releases page](https://github.com/Acly/krita-ai-diffusion/releases).

    Hover over the grayed-out entry in the Plugins Manager to see Krita's error message. Include this message when reporting an issue.
  </Accordion>

  <Accordion title="How do I update to a new version?">
    Follow the same steps as the initial installation — the old version will be replaced. Restart Krita after installation.

    If the server is managed by the plugin, it may prompt you to upgrade the server software. This reinstalls ComfyUI without re-downloading models.

    <Warning>
    `Import Python Plugin from File...` deletes the previous version. If you originally installed a version before 1.15.0, your server installation and settings may be in the plugin folder. Extract manually to `.../pykrita/ai_diffusion` instead, or relocate your server first.
    </Warning>
  </Accordion>

  <Accordion title="Where are plugin and user files located?">
    The plugin stores data in up to three locations:

    1. `<appdata>/krita/pykrita/ai_diffusion` — plugin installation folder
    2. `<appdata>/krita/ai_diffusion` — user data (settings, styles, logs)
    3. Server installation folder (configurable; defaults to the same location as user data)

    Use the "View log files" link in connection settings to find the exact path on your system.
  </Accordion>

  <Accordion title="Why is disk space required on C: drive even when installing elsewhere?">
    The system drive is used for temporary files and package manager caches during installation. You can redirect these with environment variables:

    | Cache | Default (Windows) | Environment Variable |
    |-------|-------------------|----------------------|
    | HuggingFace | `%USERPROFILE%\.cache\huggingface` | `HF_HOME` |
    | uv | `%LOCALAPPDATA%\uv\cache` | `UV_CACHE_DIR` |
    | pip | `%LOCALAPPDATA%\pip\cache` | `PIP_CACHE_DIR` |

    It is safe to delete the contents of these folders; files will be re-downloaded as needed.
  </Accordion>

  <Accordion title="How do I perform a clean re-install?">
    Re-run the standard installation — it replaces the existing plugin. To also re-install the server software (without re-downloading models), go to Connection settings → Manage → Re-install.
  </Accordion>

  <Accordion title="How do I share models from an existing Stable Diffusion installation?">
    1. Navigate to your ComfyUI server installation folder.
    2. Open `ComfyUI/extra_model_paths.yaml` in a text editor.
    3. Add or update the paths to point to your existing model directories.
  </Accordion>
</AccordionGroup>

## Connection Issues

<AccordionGroup>
  <Accordion title="Error: Could not bind on address (port already in use)">
    Another process is using port 8188. This could be:
    - Another ComfyUI instance running manually
    - A stale server process from a previous Krita session
    - An unrelated application on the same port

    **Solution:** Shut down the conflicting process, or change the server port:
    1. Open `<appdata>/krita/ai_diffusion/settings.json` in a text editor.
    2. Set `"server_arguments": "--port 25608"` (or any free port).
  </Accordion>

  <Accordion title="Connecting takes a long time or is stuck at 'Discovering models'">
    Model detection scans all checkpoint files on startup. This can be slow when:
    - You have a very large number of model files
    - Models are on a slow or network-attached drive
    - A virus scanner (e.g., Windows Defender) is scanning model files

    **Solutions:** Move models to a local SSD, archive unused models, or exclude the model folder from antivirus scanning (Windows: Settings → Virus & threat protection → Manage settings → Exclusions).
  </Accordion>

  <Accordion title="macOS: No images received from a remote ComfyUI server">
    When connecting to a remote server and images appear to generate but never appear in Krita, this is likely an SSL certificate issue on macOS. See [this issue](https://github.com/Acly/krita-ai-diffusion/issues/74#issuecomment-1819993842) for solutions.
  </Accordion>
</AccordionGroup>

## Custom ComfyUI Installation

<Note>
If you manage your own ComfyUI installation (rather than using the plugin's managed installer), you are responsible for installing prerequisites and keeping them up to date.
</Note>

<AccordionGroup>
  <Accordion title="Error: The following ComfyUI custom nodes are missing">
    Ensure the four required custom node packages are installed:

    | Package | Folder |
    |---------|--------|
    | ControlNet Preprocessors | `comfyui_controlnet_aux` |
    | IP-Adapter | `ComfyUI_IPAdapter_plus` |
    | External Tooling Nodes | `comfyui-tooling-nodes` |
    | Inpaint Nodes | `comfyui-inpaint-nodes` |

    Check that ComfyUI's startup log shows them as loaded. Also verify they appear in Comfy's web UI. **Restart ComfyUI after installing.**

    If nodes are still missing after an update, delete and reinstall the node pack — updates via ComfyUI Manager sometimes fail silently.

    <Warning>
    Running multiple ComfyUI instances simultaneously can cause the plugin to see different state than the web UI. Kill any stale `python` processes or reboot to be sure.
    </Warning>
  </Accordion>

  <Accordion title="Error: Could not find Upscale / CLIPVision / IP-Adapter model">
    Verify the model files are in the correct ComfyUI `models/` subfolder. Open ComfyUI's web UI and create the corresponding loader node to confirm ComfyUI can find them. If they are not listed, they are in the wrong folder or your `extra_model_paths.yaml` is incorrect.
  </Accordion>

  <Accordion title="Missing model: MAT Inpaint (MAT_Places512_G_fp16.safetensors)">
    Place the file in `<ComfyUI>/models/inpaint/`. Create the `inpaint` folder if it does not exist. If you use `extra_model_paths.yaml`, you must explicitly add the `inpaint` folder — it is not forwarded automatically.
  </Accordion>

  <Accordion title="Error: Hyper LoRA model not found">
    The Hyper LoRA is a required model for SD 1.5 and SDXL. Place it in the `loras/` folder. Verify it appears in a "Load Lora" node in ComfyUI's web UI.
  </Accordion>

  <Accordion title="Error while deserializing header (HeaderTooLarge / MetadataIncompleteBuffer)">
    A model file is corrupt, likely due to an incomplete download. Delete the affected file and re-download it. With the managed installer, go to Connection settings → Manage → Verify to automatically detect and re-download corrupted files.
  </Accordion>

  <Accordion title="This is a SD 1.5/XL checkpoint, but the workload has not been installed">
    Each base model requires its own set of companion models (ControlNet, IP-Adapter, etc.). If a workload is missing, you can use checkpoints of that architecture but some features will be unavailable. Install the workload via Connection settings → Manage.
  </Accordion>
</AccordionGroup>

## Generation Issues

<AccordionGroup>
  <Accordion title="Image generation is very slow">
    Check the following:
    - **Canvas resolution**: Start with a low resolution (512×512) and scale up. SD XL uses a minimum of 1024 and is inherently slower.
    - **VRAM usage**: If generation hits your VRAM limit, you may get very long generation times without an error. Use a GPU monitoring tool to check.
    - **Baseline**: Open ComfyUI's web UI at `http://127.0.0.1:8188` and run a generation directly. If performance is poor there, the issue is not specific to the plugin.
    - **Performance preset**: Use a lower performance preset (Low or Medium) in plugin settings to reduce resolution and batch size.
  </Accordion>

  <Accordion title="Windows: OSError / WinError: The specified module could not be found">
    A required Visual C++ runtime DLL is missing. Download and install the [Microsoft Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe).
  </Accordion>

  <Accordion title="Windows: [Errno 2] No such file or directory (path too long)">
    Windows has a default limit of 260 characters for file paths. This can be hit by long installation paths combined with deeply nested model files.

    **Solutions:**
    - Move the server installation to a shorter path (e.g., `C:\KritaAI`).
    - Enable long path support in Windows 10/11 (search "Enable Win32 long paths" in Group Policy Editor).
  </Accordion>

  <Accordion title="macOS: xcrun: error: invalid active developer path">
    Install Xcode command line tools:
    ```sh
    xcode-select --install
    ```
  </Accordion>
</AccordionGroup>

## Getting Help

If none of the above resolves your issue:

- [Search existing GitHub Issues](https://github.com/Acly/krita-ai-diffusion/issues?q=is%3Aissue) before opening a new one.
- Include your OS, Krita version, plugin version, GPU model, and attach the log files.
- Use [GitHub Discussions](https://github.com/Acly/krita-ai-diffusion/discussions) for questions.
- Join the [Discord server](https://discord.gg/pWyzHfHHhU) for community help.

<Warning>
Official Krita support channels are not the right place to seek help for issues specific to this plugin.
</Warning>
